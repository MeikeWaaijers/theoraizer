Welcome to the `CLDassist` package! This R package can be used to develop a Causal Loop Diagram (CLD) from scratch with the assistance of a Large Language Model (LLM). Follow the installation steps provided below to install `CLDassist` from this GitHub repository. Once installed, you can start exploring its features.

The functions in `CLDassist` and their output should be approached with caution. Depending on the specific LLM used, there may be financial implications. Furthermore, we want to emphasis that the answers generated by an LLM should not be taken as absolute truth.


## How to Use CLDassist

#### Install CLDassist

```{r eval=FALSE}
if(!require("remotes")) {
  install.packages("remotes", dependencies = T)
}

remotes::install_github("MeikeWaaijers/CLDassist")

library(CLDassist)
```

#### Quick Tryout

To quickly understand the capabilities of CLDassist, you can use the `cld()` function to generate a CLD. This function use `causal_relation()`, `causal_direction()`, `causal_sign()` and `cld_plot()` functions included in `CLDassist` with default settings. For greater control and access to additional output, it's recommended to use these functions individually.

Access the help file for further instructions.

```{r eval=FALSE}
?cld
```

#### Individual Function Use

To create a theory from scratch, the functions in this R-packaged should be used in the following order:

`var_list()` --\> `causal_relation()` --\> `causal_direction()` --\> `causal_sign()` --\> `cld_plot()` --\> `search_literature()`

Access help files for specific functions as needed.

```{r eval=FALSE}
?var_list
?causal_relation
?causal_direction
?causal_sign
?cld_plot
?search_literature
```

#### Data

Ready-made data is available to get an insight into the various outputs generated by `CLDassist`. For further details, please refer to the help files associated with these datasets.

```{r eval=FALSE}
?cld_example
?vars
?rel
?dir
?rel_sign # Data for sign function when relation_df is inputted
?dir_sign # Data for sign function when direction_df is inputted
?edge_lists
?literature
```
